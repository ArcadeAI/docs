The Arcade AI Calendar toolkit provides a pre-built set of tools for interacting with Google Calendar. These tools make it easy to build agents and AI apps that can:

- Create, update, list, and delete events

## Install

```bash
pip install arcade_google
```

Create a new event in the specified calendar.

**Parameters**

- **`summary`** _(string, required)_ The title of the event.
- **`start_date`** _(enum ([DateRange](/integrations/toolkits/Google/reference#daterange)), required)_ The day the event starts.
- **`start_time`** _(enum ([TimeSlot](/integrations/toolkits/Google/reference#timeslot)), required)_ The time the event starts.
- **`end_date`** _(enum ([DateRange](/integrations/toolkits/Google/reference#daterange)), required)_ The day the event ends.
- **`end_time`** _(enum ([TimeSlot](/integrations/toolkits/Google/reference#timeslot)), required)_ The time the event ends.
- **`calendar_id`** _(string, optional, Defaults to `'primary'`)_ The ID of the calendar.
- **`description`** _(string, optional)_ The description of the event.
- **`location`** _(string, optional)_ The location of the event.
- **`visibility`** _(string, optional)_ The visibility of the event (`'default'`, `'public'`, `'private'`, `'confidential'`).
- **`attendee_emails`** _(array of strings, optional)_ Attendee emails.

**Parameters**

- **`min_day`** _(enum ([Day](/integrations/toolkits/Google/reference#day)), required)_ Events ending on or after this day.
- **`min_time_slot`** _(enum ([TimeSlot](/integrations/toolkits/Google/reference#timeslot)), required)_ Events ending after this time.
- **`max_day`** _(enum ([Day](/integrations/toolkits/Google/reference#day)), required)_ Events starting on or before this day.

- **`calendar_id`** _(string, optional, Defaults to `'primary'`)_ The ID of the calendar.
- **`max_results`** _(integer, optional, Defaults to `10`)_ Maximum number of events to return.

## Auth

The Arcade AI Calendar toolkit uses the [Google auth provider](/integrations/auth/google) to connect to users' Google accounts.

With the hosted Arcade AI Engine, there's nothing to configure. Your users will see `Arcade AI (demo)` as the name of the application that's requesting permission.

<Warning type="warning" emoji="⚠️">
  The hosted Arcade Engine is intended for demo and testing purposes only, not
  for production use. To use Arcade AI and Google Calendar in production, you
  must use a self-hosted instance of the Arcade Engine.
</Warning>

With a self-hosted installation of Arcade AI, you need to [configure the Google auth provider](/integrations/auth/google#configuring-google-auth) with your own Google app credentials.
