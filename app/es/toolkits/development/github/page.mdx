El kit de herramientas de GitHub de Arcade proporciona un conjunto preconstruido de herramientas para interactuar con GitHub. Estas herramientas facilitan la creación de agentes y aplicaciones de IA que pueden:

- Acceder a repositorios privados (con el permiso del usuario)
- Obtener información sobre repositorios, problemas, solicitudes de extracción y más
- Publicar problemas, comentarios y respuestas como el usuario

## Github.CreateReviewComment

<br />
<TabbedCodeBlock
  tabs={[
    {
      label: "Llamar a la Herramienta con Autorización del Usuario",
      content: {
        Python: [
          "/examples/integrations/toolkits/github/create_review_comment_example_call_tool.py",
        ],
        JavaScript: [
          "/examples/integrations/toolkits/github/create_review_comment_example_call_tool.js",
        ],
      },
    },
  ]}
/>

Crear un comentario de revisión para una solicitud de extracción en un repositorio de GitHub.

Si el subject_type no es 'file', entonces los parámetros start_line y end_line son requeridos. Si el subject_type es 'file', entonces los parámetros start_line y end_line son ignorados. Si el commit_id no se proporciona, se utilizará el último SHA de commit de la rama base de la PR.

**Parámetros**

- **`owner`** _(string, requerido)_ El propietario de la cuenta del repositorio. El nombre no es sensible a mayúsculas.
- **`repo`** _(string, requerido)_ El nombre del repositorio sin la extensión .git. El nombre no es sensible a mayúsculas.
- **`pull_number`** _(integer, requerido)_ El número que identifica la solicitud de extracción.
- **`body`** _(string, requerido)_ El texto del comentario de revisión.
- **`path`** _(string, requerido)_ La ruta relativa al archivo que necesita un comentario.
- **`commit_id`** _(string, opcional)_ El SHA del commit que necesita un comentario. Si no se proporciona, se utilizará el último SHA de commit de la rama base de la PR.
- **`start_line`** _(integer, opcional)_ La línea de inicio del rango de líneas en la diferencia de la solicitud de extracción a la que se aplica el comentario. Requerido a menos que 'subject_type' sea 'file'.
- **`end_line`** _(integer, opcional)_ La línea final del rango de líneas en la diferencia de la solicitud de extracción a la que se aplica el comentario. Requerido a menos que 'subject_type' sea 'file'.
- **`side`** _(enum ([DiffSide](/toolkits/development/github/reference#diffside)), opcional, Predeterminado a `'RIGHT'`)_ El lado de la diferencia en el que aparecen los cambios de la solicitud de extracción. Usa LEFT para eliminaciones que aparecen en rojo. Usa RIGHT para adiciones que aparecen en verde o líneas sin cambios que aparecen en blanco y se muestran para contexto.
- **`start_side`** _(string, opcional)_ El lado de inicio de la diferencia a la que se aplica el comentario.
- **`subject_type`** _(enum ([ReviewCommentSubjectType](/toolkits/development/github/reference#reviewcommentsubjecttype)), opcional, Predeterminado a `'FILE'`)_ El tipo de sujeto al que se aplica el comentario. Puede ser uno de: file, hunk, o line.
- **`include_extra_data`** _(booleano, opcional, Predeterminado a `false`)_ Si es verdadero, devuelve todos los datos disponibles sobre el comentario de revisión. Esta es una carga útil grande y puede afectar el rendimiento - úsalo con precaución.

- **`per_page`** _(integer, opcional, Predeterminado a `30`)_ El número de resultados por página (máx 100).
- **`before`** _(string, opcional)_ Un cursor (identificador único, por ejemplo, un SHA de un commit) para buscar resultados antes de este cursor.
- **`after`** _(string, opcional)_ Un cursor (identificador único, por ejemplo, un SHA de un commit) para buscar resultados después de este cursor.
- **`ref`** _(string, opcional)_ La referencia de Git para las actividades que deseas listar. Puede estar formateada como `refs/heads/BRANCH_NAME` o simplemente `BRANCH_NAME`.
- **`actor`** _(string, opcional)_ El nombre de usuario de GitHub para filtrar por el actor que realizó la actividad.
- **`time_period`** _(enum ([RepoTimePeriod](/toolkits/development/github/reference#repotimeperiod)), opcional)_ El período de tiempo para filtrar.
- **`activity_type`** _(enum ([ActivityType](/toolkits/development/github/reference#activitytype)), opcional)_ El tipo de actividad para filtrar.
- **`include_extra_data`** _(booleano, opcional, Predeterminado a `false`)_ Si es verdadero, devuelve todos los datos disponibles sobre las actividades. Esta es una carga útil grande y puede afectar el rendimiento - úsalo con precaución.
