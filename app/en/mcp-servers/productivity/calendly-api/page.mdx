The CalendlyApi MCP Server provides a comprehensive set of tools for interacting with Calendly. These tools make it easy to build agents and apps that can:

- Manage and inspect users, organizations, groups, and memberships.
- List and retrieve details for event types, scheduled events, hosts, invitees, and invitee statuses (including no-shows).
- Check and manage availability: user availability schedules, busy times, and event-type available times.
- Work with organization invitations and members (create/revoke/remove), routing forms and submissions, and activity logs.
- Manage webhooks and retrieve sample webhook payloads.
- Access outgoing communications and activity/ audit data for Enterprise accounts.

<br />

      },
    },

Fetch invitees for a specific scheduled event.

**Parameters**

- **event_unique_identifier** (`string`, required) The unique identifier of the event to retrieve invitees for. Use this to specify which event's attendees you want to list.
- **invitee_status** (`string`, optional) Filter invitees by their status: 'active' or 'canceled'.
- **sort_order_by_creation** (`string`, optional) Specify the sort order of invitees by their created_at date. Use "asc" for ascending or "desc" for descending order.
- **filter_by_email** (`string`, optional) Filter results by the invitee's email address. Only invitees with this email will be returned.
- **pagination_token** (`string`, optional) Token for fetching the next or previous portion of the event invitee list.
- **number_of_invitees_to_return** (`number`, optional) The number of invitee records to return in the response. Determines how many invitees are included per request.


## CalendlyApi.ListScheduledEvents

<br />
<TabbedCodeBlock
  tabs={[
    {
      label: "Call the Tool Directly",
      content: {
        Python: ["/examples/integrations/mcp-servers/calendly_api/list_scheduled_events_example_call_tool.py"],
        JavaScript: ["/examples/integrations/mcp-servers/calendly_api/list_scheduled_events_example_call_tool.js"],
      },
    },
  ]}
/>

Retrieve a list of scheduled events from Calendly.

**Parameters**

- **user_uri** (`string`, optional) The URI identifying the user to retrieve scheduled events for. This can be used alone for personal events or with `organization` for specific user events within an organization. Requires appropriate privileges.
- **organization_uri** (`string`, optional) Returns events scheduled with the specified organization's URI. Requires admin or owner privilege.
- **invitee_email** (`string`, optional) Email address to filter events scheduled with the invitee. Use to retrieve events specific to an invitee.
- **event_status** (`string`, optional) Specifies whether to retrieve events that are 'active' or 'canceled'.
- **order_results_by** (`string`, optional) Specify the field and direction to order results (e.g., 'start_time:asc'). Supported field: start_time; Directions: asc, desc.
- **start_time_after** (`string`, optional) Include events with start times after this specified UTC time (e.g., "2020-01-02T03:04:05.678123Z").
- **max_start_time** (`string`, optional) Include events with start times prior to this UTC time. Format: "2020-01-02T03:04:05.678123Z".
- **pagination_token** (`string`, optional) Token used for paginating results, allowing access to the next or previous portion of the event collection.
- **number_of_events_to_return** (`number`, optional) Specifies the number of events to be returned in the query result.
- **group_uri** (`string`, optional) URI of the group to retrieve scheduled events for. Requires appropriate admin privileges.


## CalendlyApi.ListUserOrOrgEventTypes

<br />
<TabbedCodeBlock
  tabs={[
    {
      label: "Call the Tool Directly",

/>

Retrieve Event Types for a user or organization.

**Parameters**

- **organization_uri** (`string`, optional) The URI of the organization to view its event types. Specify this to retrieve organization event types.
- **user_uri** (`string`, optional) Specify the user's URI to view their personal, team, and organization event types.
- **user_availability_schedule_filter** (`string`, optional) Filter Event Types for a user based on the primary availability schedule. Requires the 'user' parameter to be used.
- **order_event_types_by** (`string`, optional) Order results by specified fields and directions. Use a comma-separated format: {field}:{direction}. Supported fields are: name, position, created_at, updated_at. Directions can be 'asc' or 'desc'.
- **pagination_token** (`string`, optional) Token used for paginating through results. Pass it to get the next or previous portion of the collection.
- **number_of_rows_to_return** (`number`, optional) Specify the number of event type entries to retrieve. Determines how many rows are returned in the response.
- **filter_active_event_types** (`boolean`, optional) Return only active event types if true, only inactive if false, or all event types if omitted.
- **only_admin_managed_event_types** (`boolean`, optional) Return only admin managed event types if true, exclude if false, or include all if omitted.
